Icons provide the visual context and enhance usability. Use icons to represent severity level, common actions, file types, devices, or directories.

  • Simple, intuitive icons present information that is easy to understand.
  • Product UIs should display icons with consistent style and meaning.
  • Use only whole numbers to specify icons settings (for example, dimensions, size, position coordinates, stroke width, etc.)
  • Ensure readability and clarity when resizing icons.
  • For accessibility, use color blind-friendly colors for icons.
System icons

Use system icons to represent common actions (for example, edit, delete, and search), file types, or states.
You can use system icons in toolbars or tables. System icons are usually monochrome icons. Use gray #666 for normal display and black #222 for rollover.

Severity icons

Use severity icons to indicate risk levels (for example, safe, low, medium, and high) or device states (error, warning, success).

Graphic icons

Use graphic icons to communicate feature concept or product service. Graphic icons help to enhance user memory and are often used on the Dashboard or What's New page.
